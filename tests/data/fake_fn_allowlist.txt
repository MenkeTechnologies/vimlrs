# fake_fn_allowlist.txt — sanctioned `fn` names under src/ported/ that are NOT
# in the csrc-derived list (docs/nvim_c_functions.txt).
#
# Ported from zshrs's tests/data/fake_fn_allowlist.txt + ported_fn_names_match_c.
# The test `tests/ported_fn_names_match_c.rs` rejects any fn under src/ported/
# whose name is neither in docs/nvim_c_functions.txt NOR listed here NOR a
# trait-impl / #[test] exemption. Adding a name here is the audit trail for an
# exception — keep each justified.
#
# Every entry below is a REAL Neovim function whose defining C file is not
# vendored under csrc/ (only the eval/ tree is). They are faithful ports of real
# Vim builtins/helpers, cited to their home file — analogous to the already-
# recognized extern deps emsg/vim_str2nr (those appear as calls in the eval tree,
# so they land in the generated list; these do not). There are NO synthesis
# adapters here: those have been inlined or expressed as `From`/trait impls.
# When math.c / strings.c / option.c get vendored, move their names out of this
# file and the generator will recognize them.
